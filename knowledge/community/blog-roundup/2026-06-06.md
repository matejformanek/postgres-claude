## 1. Looking Forward to Postgres 19: Query Hints — Shaun Thomas

URL: https://postgr.es/p/9ln · 2026-06-05

- Walks the decades-long pgsql-hackers resistance to query hints and explains
  how PG19's **`pg_plan_advice`** lands a deliberately-scoped alternative —
  plan *advice* (shape steering) rather than classic per-query hint syntax.
- Frames why the design sidesteps the historical "hints rot / hide planner
  bugs" objection.
- **Read if you care about:** the planner, plan stability, and the political
  history behind why PG never shipped Oracle-style hints. cf.
  [[knowledge/architecture/planner.md]], [[knowledge/subsystems/optimizer.md]].

## 2. Beta Testing PostgreSQL With Docker — Andrew Atkinson

URL: https://postgr.es/p/9lr · 2026-06-05

- Hands-on guide to running **PG19 Beta 1 in Docker** to exercise new features,
  calling out `pg_plan_advice` and the enhanced `pg_stat_statements`.
- Practical recipe: pull the beta image, spin up, poke the new views.
- **Read if you care about:** kicking the tires on PG19 beta locally before it
  hits your distro; pairs with the corpus's pg_stat_statements work. cf.
  [[knowledge/issues/pg_stat_statements.md]].

## 3. How to test PostgreSQL 19 beta in your Kubernetes cluster — Floor Drees

URL: https://postgr.es/p/9li · 2026-06-05

- Walks testing PG19 beta on Kubernetes via the **CloudNativePG** operator.
- Operator-managed approach for teams that run PG on k8s and want beta coverage
  in a realistic orchestration setting.
- **Read if you care about:** CloudNativePG, beta validation in k8s, operator
  upgrade paths. `[inferred]`

## 4. File Descriptors: The OS Limit That Takes Down PostgreSQL — warda bibi

URL: https://postgr.es/p/9lb · 2026-06-04

- Explains how **file-descriptor exhaustion** causes PostgreSQL outages, with
  diagnosis (ulimit, `lsof`, per-backend fd accounting) and remediation.
- Timely: directly mirrors this week's hackers thread *"File descriptor leak
  when io_method=io_uring"* — the io_uring AIO backend is a concrete in-tree
  source of exactly this failure mode.
- **Read if you care about:** backend resource limits, AIO/io_uring fd
  lifecycle, production stability. cf.
  [[knowledge/community/threads/2026-06-06.md]],
  [[knowledge/subsystems/storage-buffer.md]].

## 5. All Your GUCs in a Row: data_checksums — Christophe Pettus

URL: https://postgr.es/p/9lc · 2026-06-05

- Covers the **`data_checksums`** read-only GUC and its ~thirteen-year road from
  proposal to default-on, including why it's read-only at runtime.
- Companion same-series post **data_directory** (https://postgr.es/p/9lq,
  2026-06-06) covers the bootstrapping puzzle of locating config files relative
  to PGDATA.
- **Read if you care about:** page-level corruption detection, the
  cluster-init/`initdb` checksum decision, and GUC semantics. cf.
  [[knowledge/idioms/guc-variables.md]],
  [[knowledge/files/src/bin/initdb/initdb.c.md]].
