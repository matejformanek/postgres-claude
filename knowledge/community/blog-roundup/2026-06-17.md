## 1. All Your GUCs in a Row: default_transaction_deferrable

- Author: Christophe Pettus · 2026-06-17 · https://postgr.es/p/9ma
- Walks `default_transaction_deferrable`, the GUC that makes new transactions
  deferrable by default — relevant only for SERIALIZABLE READ ONLY
  transactions, where it lets the SSI machinery wait for a safe snapshot
  instead of risking a serialization-failure rollback.
- Practical note: it's a no-op outside serializable read-only, so the GUC is
  narrower than its name suggests.

**Read if you care about:** SSI / serializable isolation, the deferrable-snapshot
optimization. [[knowledge/idioms/snapshot-static-and-current.md]]

## 2. Releasing pg_ducklake v1.0

- Author: Qiaosheng Liu · 2026-06-17 · https://postgr.es/p/9mc
- Announces a 1.0 of pg_ducklake, a PostgreSQL extension exposing
  lakehouse-style (DuckDB/Parquet-backed) storage, with claimed ingestion
  performance improvements over the pre-1.0 line.
- Of internals interest: how it bridges PG's table-AM / FDW surface to an
  external columnar store.

**Read if you care about:** table access methods, columnar/lakehouse
integration, FDW vs. custom-AM design. [[knowledge/idioms/tableam-index-fetch.md]]

## 3. pgstream v1.1.0: Steps towards turning it into a service

- Author: Noémi Ványi · 2026-06-16 · https://postgr.es/p/9m5
- pgstream (logical-replication-driven CDC/streaming) 1.1.0 adds env-var
  interpolation in YAML config, JSON structured logging, and Kubernetes
  compatibility — operational steps toward running it as a managed service.
- Built on PG logical decoding output; the service framing is the headline.

**Read if you care about:** logical replication / CDC tooling, running
decoding-based pipelines in production. [[knowledge/architecture/replication.md]]
