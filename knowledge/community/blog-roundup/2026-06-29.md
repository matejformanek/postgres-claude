## 4. Radim Marek — Same rows, different SUM

- <https://postgr.es/p/9nx> · 2026-06-28
- Parallel aggregation over floating-point columns yields non-deterministic
  results: IEEE-754 addition isn't associative, so worker partition boundaries
  change the rounding; recommendation is to use `numeric` when determinism
  matters.
- Directly mirrors today's pgsql-hackers stats-precision threads (floating-point
  noise in `pg_stat_us_to_ms`, parallel-agg ordering) — the same root cause
  surfacing in user-visible query results.
- **Read if you care about:** parallel query correctness, float vs numeric,
  aggregate determinism. [[knowledge/idioms/aggregate-partial-finalize.md]],
  [[knowledge/idioms/cost-parallel-adjustments.md]]

## 8. Lætitia AVROT — Stop Punishing Your Postgres for a Crash That Won't Happen

- <https://postgr.es/p/9nu> · 2026-06-27

  tradeoff; touches the checkpointer/WAL-volume relationship.
- **Read if you care about:** checkpoint tuning, WAL volume, crash-recovery
