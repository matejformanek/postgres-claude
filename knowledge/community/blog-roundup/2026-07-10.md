## 1. The Version Number Is Not the Territory

- **Author:** Christophe Pettus
- **URL:** <https://postgr.es/p/9pe>
- **Published:** 2026-07-09 20:00 UTC
- TL;DR:
  - Walks through a puzzling case where a PostgreSQL 14 instance threw an error
    that "shouldn't be possible" in that major version — a reminder that the
    reported version string doesn't fully pin down the actual behavior you're
    running against (extensions, minor-version fixes, backports).
  - The lesson: diagnose from observed behavior and actual binaries, not from the
    `server_version` label alone.
- **Read if you care about:** debugging version-specific behavior, minor-release
  backports, why "it can't happen in 14" is a dangerous assumption.

## 2. PostgreSQL, AI Governance, and the C.A.L.M. Platform Test

- **Author:** Vibhor Kumar
- **URL:** <https://postgr.es/p/9pc>
- **Published:** 2026-07-09 16:40 UTC
- TL;DR:
  - Proposes a four-pillar framework — **C**hangeability, **A**ssurance,
    **L**everage, **M**easurability — for enterprise AI governance grounded in the
    database layer rather than the model layer.
  - Argues PostgreSQL's native features (logical replication for change capture,
    Row Level Security for access assurance, pgvector for retrieval) are the
    substrate that makes AI systems auditable and controllable.
- **Read if you care about:** RLS and logical replication as governance
  primitives, pgvector in enterprise architectures, positioning PG as AI data
  infrastructure.

## 4. How to Achieve Pruning When Querying by Non-Partitioned Columns in PostgreSQL

- **Author:** Haki Benita
- **URL:** <https://postgr.es/p/9pa>
- **Published:** 2026-07-09 12:41 UTC
- TL;DR:
