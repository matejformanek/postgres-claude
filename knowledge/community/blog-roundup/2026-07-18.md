## New this window (last 24h)

### 1. Christophe Pettus — All Your GUCs in a Row: extension_control_path
- **URL:** https://postgr.es/p/9pY · Sat 18 Jul 2026 01:00 UTC
- Latest entry in Pettus's per-GUC series: **`extension_control_path`** (a
  PG18 addition) lets you place an extension's `.control` file and SQL scripts
  **outside** the hard-coded `SHAREDIR/extension` directory, so an extension
  no longer has to be installed into system directories to be `CREATE
  EXTENSION`-able.
- TL;DR:
  - The GUC is a colon-separated search path (like `dynamic_library_path` for
    the `.so`), with a `$system` token expanding to the built-in dir — so you
    prepend a user-writable location and keep system extensions reachable.
  - Practical payoff: per-user / per-project extension trees, immutable-OS and
    container images where `SHAREDIR` is read-only, and testing an extension
    build without `sudo make install`.
- **Read if you care about:** extension packaging/deployment, PG18 operational
  changes, immutable/containerized PG images. [[extension-development]] ·
  [[gucs-config]].
