## 1. When Patroni Silently Deletes Your Replication Slots
- **Author:** semab tariq · 2026-06-12 · https://postgr.es/p/9lM
- Patroni reconciles the slot set on restart and **drops replication slots it
  doesn't recognize** — which can silently delete slots you created out-of-band
  (e.g. for a migration or an external CDC consumer).
- The `ignore_slots` config tells Patroni to leave named slots alone; set it
  before a migration that depends on a hand-made slot.
- **Read if you care about:** HA orchestration, logical replication slots,
  failover slot management. cf. [[knowledge/subsystems/replication.md]].

## 2. Looking Forward to Postgres 19: It's About Time
- **Author:** Shaun Thomas · 2026-06-12 · https://postgr.es/p/9lL
- PG19's headline is **native temporal-table support**: range-typed period
  columns, automatic row-splitting on temporal updates, and temporal foreign
  keys (`PERIOD` / `WITHOUT OVERLAPS`).
- Frames temporal tables as the SQL:2011 application-time feature finally landing
  in core rather than living in extensions.
- **Read if you care about:** PG19 feature scope, temporal/bitemporal modeling,
  range types. cf. [[knowledge/subsystems/access-heap.md]].

## 3. All Your GUCs in a Row: default_statistics_target
- **Author:** Christophe Pettus · 2026-06-12 · https://postgr.es/p/9lK
- Raising `default_statistics_target` enlarges the MCV list + histogram the
  planner sees, improving estimates on skewed columns — but **slows ANALYZE** and
  inflates `pg_statistic`, so it's a targeted (per-column) tool, not a global
  dial to crank.
- Latest in Pettus's GUC-by-GUC series; pairs with the planner-internals post (#6).
- **Read if you care about:** planner statistics, ANALYZE cost, query tuning. cf.
  [[knowledge/subsystems/optimizer.md]].

## 4. A thousand Postgres branches for $1
- **Author:** Tudor Golubenco · 2026-06-11 · https://postgr.es/p/9lI
- Copy-on-write database branching for dev/CI: engineering work cut branch
  creation from **~20 s to ~1 s**, making per-PR ephemeral databases cheap enough
  to spin up by the thousand.
- The speedup is storage-layer (CoW snapshot plumbing), not a Postgres-core
  change — relevant as a pattern for disposable test clusters.
- **Read if you care about:** dev/test tooling, CoW storage, CI database
  provisioning.

## 5. Buildfarm Query API
- **Author:** Andrew Dunstan · 2026-06-11 · https://postgr.es/p/9lJ
- A new **programmatic query API** over the PostgreSQL Buildfarm database
  replaces the old scrape-the-HTML approach for pulling build/test results.
- Lets tooling correlate failures across animals/branches without screen-scraping
  — useful for triaging flaky TAP/isolation failures across platforms.
- **Read if you care about:** CI infrastructure, buildfarm triage, test-failure
  analytics. cf. [[knowledge/community/threads/2026-06-12.md]] (the
  `035_standby_logical_decoding.pl` timeline flake).

## 6. How the Other Half Plans
- **Author:** Christophe Pettus · 2026-06-11 · https://postgr.es/p/9lH
- A planner-internals explainer: query planning is **rewrite + cost-based search**
  over a combinatorially large join-order space, pruned by cost estimates rather
  than enumerated exhaustively.
- Good lay-of-the-land companion to the `default_statistics_target` post (#3) —
  why good stats matter to the search.
- **Read if you care about:** the optimizer's join-order search, cost model
  intuition. cf. [[knowledge/subsystems/optimizer.md]].
