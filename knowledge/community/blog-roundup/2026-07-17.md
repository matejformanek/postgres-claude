## 1. Looking Forward to Postgres 19: Checkpoint Control — Shaun Thomas

- **URL:** https://postgr.es/p/9pX · Fri 17 Jul 2026 06:05 UTC
- TL;DR:
  - Walks the new PG19 checkpoint-control surface — `MODE` and `UNLOGGED`
    settings that let a DBA tune how aggressively a manual/triggered
    checkpoint flushes, trading I/O spike vs. duration.
  - Frames it as giving operators a knob between the "flush everything now"
    hammer and the smoothed background writer.
- **Read if you care about:** checkpoint tuning, `CHECKPOINT` command options,
  I/O-spike smoothing on PG19. Pairs with corpus WAL/checkpoint material.

## 2. All Your GUCs in a Row: exit_on_error — Christophe Pettus

- **URL:** https://postgr.es/p/9pW · Fri 17 Jul 2026 01:00 UTC
- TL;DR:
  - Dissects `exit_on_error`: when on, any error terminates the session,
    which sounds like defensive hygiene but is usually a foot-gun (one bad
    query kills the whole connection).
  - Recommends leaving it off outside narrow batch/ETL scenarios.
- **Read if you care about:** GUC semantics, session-lifecycle error handling.
  Timely against hackers thread #11 (recovery-target assign-hook `ereport`) —
  both about where an error is allowed to terminate vs. reject.

## 3. Spock 6: The Only Logical Choice — Antony Pegg

- **URL:** https://postgr.es/p/9pV · Thu 16 Jul 2026 20:16 UTC
- TL;DR:
  - pgEdge's Spock multi-master logical-replication extension hits beta 6:
    shared-memory progress tracking, spill-to-disk for large transactions,
    and conflict detection aligned to PG18's built-in conflict-tracking.
  - Positions Spock as the multi-master option now that PG18 shipped native
    conflict logging.
- **Read if you care about:** logical replication internals, multi-master
  conflict resolution, PG18 conflict-tracking reuse by extensions.
