## 1. Query Hints in PostgreSQL 19 — Shaun Thomas (2026-06-05)

`https://postgr.es/p/9ln`

- After two decades of resistance, PG19 ships query hints via **two contrib
  modules** — `pg_plan_advice` and `pg_stash_advice` — that *constrain* the
  planner's search space rather than overriding its cost model.
- Framed as a deliberate compromise: hints narrow which paths the planner may
  consider, preserving costing/correctness instead of pinning a fixed plan.
- **Read if you care about:** the planner, plan stability, and why core
  finally accepted a hinting mechanism. cf.
  [[knowledge/subsystems/optimizer.md]], [[knowledge/architecture/planner.md]].

## 2. File Descriptors: The OS Limit That Takes Down PostgreSQL — warda bibi (2026-06-04)

`https://postgr.es/p/9lb`

- FD exhaustion under high connection counts gets misdiagnosed as a "database
  problem"; each backend is a process holding its own FDs, so connection count
  multiplies open handles fast.
- Recommended fix is a connection pooler (PgBouncer) ahead of raising OS
  `ulimit` / `max_files_per_process`.
- **Read if you care about:** the per-connection process model and its OS-level
  resource footprint. cf. [[knowledge/architecture/process-model.md]].

## 3. Does pgBackRest Work with pg_tde? — Stefan Fercot (2026-06-04)

`https://postgr.es/p/9la`

- pgBackRest handles `pg_tde`-encrypted clusters transparently — async WAL
  archiving and restore work despite earlier claims to the contrary.
- Relevant as transparent data encryption (pg_tde) moves toward mainstream
  backup-tool support.
- **Read if you care about:** TDE, WAL archiving, and backup/restore of
  encrypted clusters. cf. [[knowledge/architecture/wal.md]].

## 4. Handling Graphs with SQL/PGQ in PostgreSQL — Hans-Juergen Schoenig (2026-06-03)

`https://postgr.es/p/9l7`

- PG19's SQL/PGQ support adds `CREATE PROPERTY GRAPH` + `GRAPH_TABLE` for
  querying graph structure over plain relational tables, no extension needed.
- Mirrors the active pgsql-hackers GRAPH_TABLE / SQL/PGQ bug threads this same
  window (lateral label-disjunction, orphaned-label cleanup).
- **Read if you care about:** the SQL/PGQ parser+executor surface landing in
  PG19. cf. [[knowledge/subsystems/parser-and-rewrite.md]],
  [[knowledge/subsystems/executor.md]].

`https://postgr.es/p/9l8`
