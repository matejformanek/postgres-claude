# Planet PostgreSQL — blog roundup, 2026-06-20

Source: `https://planet.postgresql.org/rss20.xml`. Fetched 2026-06-20.
`[from-website]`

Only 4 posts fell strictly in the last 24h (06-19/06-20), below the 6-10 target,
so the window is stretched back to 06-17 to fill. 8 posts selected, internals-
and feature-relevant ones prioritized over pure AI/marketing pieces.

---

1. **All Your GUCs in a Row: dynamic_shared_memory_type** — Christophe Pettus
   (06-20). https://postgr.es/p/9mz
   - Walks the `dynamic_shared_memory_type` GUC: why parallel query needs DSM
     allocated at *runtime* (per-query segments) rather than at postmaster start.
   - Covers the `posix`/`sysv`/`mmap` backends and when you'd ever change it.
   - *Read if you care about:* parallel query, DSM/shm_toc, the
     `[[knowledge/skills:parallel-query]]` path.

2. **Looking Forward to Postgres 19: Logically Sequenced** — Shaun Thomas
   (06-19). https://postgr.es/p/9mu
   - PG19 finally replicates *sequence* values over logical replication, closing
     a decade-old gap that caused duplicate-key errors after failover/migration.
   - Means convert-to-subscriber migrations no longer need a manual sequence sync.
   - *Read if you care about:* logical replication, zero-downtime migration —
     directly pairs with today's hackers thread "Include sequences in
     publications created by pg_createsubscriber" (threads §10).
     [[knowledge/architecture/replication.md]]

3. **All Your GUCs in a Row: dynamic_library_path** — Christophe Pettus (06-19).
   https://postgr.es/p/9mr
   - How PG18's `extension_control_path` (atop `dynamic_library_path`) made
     extensions truly relocatable, decoupling them from the fixed `$libdir`.
   - *Read if you care about:* extension packaging, multi-version install layouts.

4. **It's Not Magic, It's Method** — Lætitia AVROT (06-19). https://postgr.es/p/9ms
   - A methodology for evaluating PG extensions by reading their source and
     understanding the implementation well enough to fix bugs yourself.
   - *Read if you care about:* extension due-diligence, contributing back.

5. **All Your GUCs in a Row: default_transaction_isolation and
   default_transaction_read_only** — Christophe Pettus (06-18).
   https://postgr.es/p/9me
   - Tours the isolation-level GUCs and flags some counterintuitive per-session
     vs per-transaction interaction in how the defaults apply.
   - *Read if you care about:* MVCC isolation semantics, SSI. [[knowledge/architecture/mvcc.md]]

6. **Introducing ColdFront: Uniting OLTP, Analytics and AI Workloads on
   PostgreSQL** — Antony Pegg / pgEdge (06-18). https://postgr.es/p/9mf
   - pgEdge ColdFront beta: open-source transparent data tiering with a "fully
     writable cold tier" for PostgreSQL.
   - *Read if you care about:* storage tiering, HTAP, table-AM-adjacent extensions.

7. **Releasing pg_ducklake v1.0** — Qiaosheng Liu (06-17). https://postgr.es/p/9mc
   - Production 1.0 of pg_ducklake: broad DuckLake coverage and a fast ingestion
     path turning PG into a lakehouse front-end.
   - *Read if you care about:* lakehouse/DuckDB integration, analytics offload.

8. **All Your GUCs in a Row: default_transaction_deferrable** — Christophe Pettus
   (06-17). https://postgr.es/p/9ma
   - Explains DEFERRABLE transactions — only meaningful for SERIALIZABLE READ
     ONLY transactions, where they wait for a safe snapshot to avoid SSI aborts.
   - *Read if you care about:* serializable isolation, predicate locks.
     [[knowledge/idioms/predicate-locks.md]]
