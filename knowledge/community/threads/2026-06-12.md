### 1. Commit Sequence Numbers and Visibility
- **Author:** Matthias van de Meent · active 2026-06-12 10:03. `[from-website]`
- A CSN-based visibility scheme replaces (or augments) the snapshot-as-XID-array
  model with a monotonic commit sequence number stamped at commit; visibility
  becomes a CSN comparison rather than a linear scan of an in-progress XID set.
  `[inferred]`
- The recurring hard parts are the same ones every CSN proposal hits: where the
  CSN is stored for an in-progress vs. committed xact, how `clog`/`xact` slots
  map to CSNs, and how this interacts with hot-standby snapshot export. `[inferred]`
- **Why it matters:** CSN visibility is the perennial "rewrite the MVCC core"
  proposal; it touches snapshot acquisition cost on big-`max_connections` boxes.
  cf. [[knowledge/data-structures/snapshot-lifecycle.md]],
  [[knowledge/subsystems/access-transam.md]], [[knowledge/subsystems/access-heap.md]].

### 2. Fix race in ReplicationSlotRelease for ephemeral slots
- **Author:** Xuneng Zhou · Amit Kapila reply 2026-06-12 10:54. `[from-website]`
- An ephemeral (`RS_EPHEMERAL`) slot dropped on release has a window where a
  concurrent backend can observe the slot mid-teardown; the fix tightens the
  order of in-use flag clearing vs. the actual slot free under
  `ReplicationSlotControlLock`. `[inferred]`
- Ephemeral slots are the throwaway slots used by base-backup/`CREATE
  SUBSCRIPTION` setup; the race is narrow but corrupts the slot array if hit. `[inferred]`
- **Why it matters:** slot-array races are exactly the class flagged in the CF
  internals baseline (#6451 slot-min-LSN race). cf.
  [[knowledge/subsystems/replication.md]], [[knowledge/subsystems/storage-ipc.md]].

### 3. synchronized_standby_slots behavior inconsistent with quorum-based synchronous replication
- **Author:** Shlok Kyal · replies from Ashutosh Sharma 2026-06-12 10:10–13:03. `[from-website]`
- `synchronized_standby_slots` is meant to hold back logical slot advance until
  named physical standbys confirm, but its all-of-the-list semantics don't line
  up with quorum (`ANY k (...)`) `synchronous_standby_names`: a quorum commit can
  release while a non-quorum standby in the slot list hasn't caught up. `[inferred]`
- The question under debate is whether the slot-sync wait should mirror the
  quorum set or stay strict-AND over the listed slots. `[inferred]`
- **Why it matters:** failover-slot correctness under quorum sync rep is a
  data-loss-on-failover concern, not cosmetic. cf.
  [[knowledge/subsystems/replication.md]].

### 4. [PATCH] Fix PITR pause bypass when initial XLOG_RUNNING_XACTS has subxid overflow
- **Author:** Jan Nidzwetzki · 2026-06-12 09:59. `[from-website]`
- When the first `XLOG_RUNNING_XACTS` record seen during recovery has the
  subxid-overflow flag set, the recovery-target pause logic can be skipped, so a
  `recovery_target_*` + `pause` configuration overshoots its target. `[inferred]`
- Root cause is consistency/snapshot-state not yet being established when the
  overflowed running-xacts record arrives, so the pause check evaluates against
  an incomplete known-assigned-xids set. `[inferred]`
- **Why it matters:** silently blowing past a PITR target undermines the one
  guarantee PITR exists to give. cf. [[knowledge/subsystems/access-transam.md]].

### 5. Deadlock detector fails to activate on a hot standby replica
- **Author:** Fujii Masao · Vitaly Davydov 2026-06-12 06:00, Fujii 09:23
  (continuing from earlier this week). `[from-website]`
- On a hot standby, recovery conflicts and lock waits between the startup process
  and read-only backends can wedge without the deadlock detector firing, because
  the detector's timer path isn't armed the same way on the standby. `[inferred]`
- **Why it matters:** a standby that can deadlock-without-detection stalls read
  replicas indefinitely. cf. [[knowledge/subsystems/storage-lmgr.md]],
  [[knowledge/idioms/locking-overview.md]].

### 6. injection_points: Switch wait/wakeup to use atomics rather than latches
- **Author:** Andrey Borodin · 2026-06-12 07:02. `[from-website]`
- The `injection_points` test framework's wait/wakeup currently rides on the
  latch infrastructure; switching to an atomic flag + spin removes a dependency
  on a backend's latch being free and makes the wait point usable in code paths
  that themselves manipulate latches. `[inferred]`
- **Why it matters:** injection points are how concurrency races (like #2 and #3
  above) get deterministic isolation tests; broadening where they can be placed
  improves the whole test surface. cf. [[knowledge/idioms/locking-overview.md]],
  [[knowledge/data-structures/pgproc-fields.md]].

### 7. Fix ALTER DOMAIN VALIDATE CONSTRAINT locking
- **Author:** Chao Li · 2026-06-12 14:01, with Fujii Masao (11:47) and Álvaro
  Herrera (13:02) on the thread. `[from-website]`
- `ALTER DOMAIN ... VALIDATE CONSTRAINT` appears to take a weaker lock than the
  validation scan needs, leaving a window where concurrent DML against columns of
  the domain type races the constraint flip. `[inferred]`
- The fix is about picking the correct lock level on the domain (and dependent
  relations) for the validate path. `[inferred]`
- **Why it matters:** wrong lock level on a catalog-mutating DDL is a classic
  source of "constraint marked valid but a violating row slipped in" bugs. cf.
  [[knowledge/subsystems/storage-lmgr.md]], [[knowledge/idioms/catalog-conventions.md]].

### 8. REPACK CONCURRENTLY fails on tables with generated columns

- `REPACK CONCURRENTLY` (the PG19 online-rewrite command) chokes when the target

- **Why it matters:** REPACK CONCURRENTLY is a headline PG19 feature; a

  [[knowledge/subsystems/access-heap.md]], [[knowledge/subsystems/executor.md]].

### 9. [PATCH] Fix replica identity mismatch for partitioned tables with publish_via_partition_root
- **Author:** Alexander Korotkov · 2026-06-12 05:46. `[from-website]`
- With `publish_via_partition_root`, changes are published as the root's row
  shape; if a leaf partition's replica identity doesn't match the root's, the
  decoded change carries the wrong key columns and the subscriber's apply fails
  or mis-targets a row. `[inferred]`
- **Why it matters:** silent key mismatch in logical replication causes
  divergence between publisher and subscriber. cf.
  [[knowledge/subsystems/replication.md]], [[knowledge/subsystems/partitioning.md]].

### 10. Add "FOR UPDATE NOWAIT" lock details to the log
- **Author:** Fujii Masao · 2026-06-12 07:27. `[from-website]`
- Proposes logging the blocking-PID / lock-tag detail when a `SELECT ... FOR
  UPDATE NOWAIT` bails out with `lock_not_available`, mirroring the

- **Why it matters:** NOWAIT failures are currently opaque ("could not obtain
