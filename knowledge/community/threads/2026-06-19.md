## 2. Deadlock detector fails to activate on a hot standby replica
- **Author:** Fujii Masao · Vitaly Davydov (active again; latest 06-18 16:22)
- **URL:** https://www.postgresql.org/message-id/CAHGQGwH325%2B6qz0g50XH-MrLe7xNH1d9snEqXevdCR3ACnLJoQ%40mail.gmail.com
- On a standby the `deadlock_timeout` path doesn't arm for backends blocked on
  recovery-conflict / buffer-pin waits, so a genuine wait cycle can hang past
  the timeout without `CheckDeadLock` ever running.
- Davydov's reply distinguishes the startup (recovery) process's wait handling
  from a normal backend's detector arming and proposes where to hook the timer.
- **Why it matters:** the silent-hang failure mode on the replica side — the
  class the locking corpus warns about (`wait_event_type='LWLock'` but no
  "deadlock detected" ever fires). [[knowledge/idioms/deadlock-detection.md]]

## 4. Prove a NOT IN's left-hand expressions non-nullable from quals
- **Author:** Richard Guo (NEW thread, 06-19 01:13)
- **URL:** https://www.postgresql.org/message-id/CAMbWs4_TNUs1jn7q0J-%3DEsz7ziiFdjDAtW4x2u6tv6H5hhmhDA%40mail.gmail.com
- Proposes that the planner prove a `NOT IN (subquery)`'s left-hand expressions
  are non-nullable from existing quals, so the anti-join can use a cheaper plan
  instead of the conservative NULL-aware semijoin handling.
- New patch with a worked example showing the plan improvement when a
  `WHERE col IS NOT NULL` (or implied) qual is present.
- **Why it matters:** `NOT IN` is a long-standing planner footgun precisely
  because of NULL semantics; tightening the non-null proof removes a real
  performance cliff for a common query shape. [[knowledge/subsystems/optimizer.md]]

## 7. Fix race in ReplicationSlotRelease for ephemeral slots
