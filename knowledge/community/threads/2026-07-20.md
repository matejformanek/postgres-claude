## 1. read stream: Backward I/O combining
- **Author(s):** Nazir Bilal Yavuz (new thread), Tomas Vondra (reply)
- **URL:** `/message-id/CAN55FZ1nB1p_mqnzVY1_CA-OXgQD1yTEJdHXX4E1hDZxVHQRQA%40mail.gmail.com`
- Extends the read-stream I/O-combining machinery to **backward** scan
  direction — today's combining only coalesces forward-adjacent blocks, so a
  backward index or heap scan issues one I/O per block.
- Tomas Vondra engaged same-day (14:57), so this has committer-adjacent
  attention rather than sitting cold.
- Lands squarely on the PG 17/18 read-stream API surface; a new consumer
  shape (descending block ranges) that the callback contract has to express.
- **Why it matters:** backward-scan prefetch is a known gap in the AIO story;
  this is the concrete patch that closes it. `[[knowledge/idioms/read-stream-prefetch.md]]`,
  `[[.claude/skills/aio-readstream]]`.

## 3. Checkpointer write combining
- **Author:** Andrey Borodin
- **URL:** `/message-id/D0E7C430-76A9-4030-B8AB-109E7D7ED8C1%40yandex-team.ru`
- Combining adjacent dirty-buffer writes at checkpoint time to cut the number

## 8. pg_class.reltuples can become non-finite and never recovers
- **Author(s):** Jan Nidzwetzki, Tomas Vondra
- **URL:** `/message-id/518BA772-8026-412A-AA8F-A7FE4C6B3717%40planetscale.com`
- A live bug report: `pg_class.reltuples` can be driven to a non-finite

- **Why it matters:** a stuck non-finite reltuples silently wrecks every plan

  `[[knowledge/subsystems/executor.md]]` (planner cardinality inputs).
