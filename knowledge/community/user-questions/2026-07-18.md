### 1. Transaction Isolation level for ERP software
- URL: `/message-id/CADWR%2BqdOYKvq6WCUsB8bs%2BV_5DL-qX8B_b%3DEbHiatvOimCfaYg%40mail.gmail.com`
- Source: pgsql-general
- Summary: A user building ERP software asks which isolation level to
  standardize on (Read Committed vs Repeatable Read vs Serializable) and how
  PG's MVCC snapshot semantics affect lost-update / write-skew anomalies in
  their transaction mix. Classic "which level do I actually need" question
  grounded in PG's snapshot model.
- **Skill tag:** `snapshot-management`
- Ideal answer outline: walk the three levels via
  `knowledge/architecture/mvcc.md` + `snapshot-management` skill ‚ÄĒ
  Read Committed takes a fresh snapshot per statement, RR/Serializable one
  per-transaction (`GetTransactionSnapshot`); SSI predicate locks catch
  write-skew only at Serializable.
