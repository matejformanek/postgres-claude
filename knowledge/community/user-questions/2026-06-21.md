## 3. synchronized_standby_slots behavior inconsistent with quorum-based synchronous replication

- URL: https://www.postgresql.org/message-id/CAJpy0uDnx2YX_oHLPP-xsO9qbdktoNtGg2dF601YpxdH%2BXHSKw%40mail.gmail.com
- List: pgsql-hackers
- Summary: shveta malik observes that `synchronized_standby_slots` (which gates
  logical-slot advancement on physical standbys having confirmed the LSN) does
  not compose correctly with quorum-based `synchronous_standby_names` (ANY k):
  the logical side waits on *all* listed slots while the sync-rep side only needs
  a quorum, so logical decoding can stall behind a standby that quorum sync never
  required.
- **Skill tag:** `replication-overview`
- Ideal answer: contrast `StandbySlotsHaveCaughtup` / `WalSndWaitForWal` with the
  quorum logic in `syncrep.c` (`SyncRepGetSyncStandbysQuorum`); cite the
  replication subsystem docs for the failover-slot + sync-rep interaction.
