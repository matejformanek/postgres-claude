### 1. crake — RedisFDW-installcheck-C — all branches — **external module, not PG** ⭐

- Animal: `crake`, Fedora 44, gcc 16.1.1. Branches: master `[e051188]`,
  REL_18 `[b1ab4bc]`, REL_17 `[e4183c6]`, REL_16 `[8d516d2]`, REL_15 `[0f63b74]`,
  REL_14 `[1b79c8d]` — and again at the next SHA on each branch.
- Stage: `RedisFDW-installcheck-C`. History: `show_history.pl?nm=crake&br=master`.
- **Rootcause (high confidence):** `RedisFDW` is a **third-party FDW** that crake
  builds as an extra module against every PG branch. The stage fails on *all six
  branches at every run*, which cannot be caused by any single core commit — it's
  the external module (or its Redis test dependency / server availability) that's
  broken or incompatible with current crake. Core PG is not implicated.
- Lesson: animal-specific extra-module stages (`RedisFDW-*`, other `*FDW-*`)
  failing uniformly across branches are **noise for upstream-watching** — filter
  them out unless the same stage starts failing on animals that don't run the
  external module.
