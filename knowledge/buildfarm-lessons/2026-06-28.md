## Root-cause notes

### `koel` — `indent-check` red — **GENUINE SIGNAL** `[verified-by-code]`

`koel` (Fedora 44) carries the system note "Runs pgindent --show-diff", so its
`indent-check` stage fails whenever a committed C file isn't pgindent-clean.
The failing snapshot `f03ecd2` (22:49Z) lists `src/backend/commands/copyto.c`
among "Files changed this run" — confirmed in the fetched log. That is the
`effb923d9de` ("COPY TO FORMAT JSON: respect column list order", 20:48Z) hunk,
which was not re-indented. Andrew Dunstan pushed the cosmetic fix
`02f699c141` ("pgindent fix for commit effb923d9de") at 23:57Z, ~1h after
koel's run. **This is the buildfarm working exactly as intended**: a
pgindent-checking animal flagged a style regression on master within the hour,
and it was self-healed before the next nightly cycle. No corpus action beyond
noting it as a textbook `indent-check` case — reinforces the
`[[knowledge/subsystems/...]]` coding-style discipline (pgindent must run
before commit; the project's own pre-commit hook enforces the same gate).
Fingerprint: `koel` + `indent-check` + a C file in "Files changed this run" =
real pgindent violation, expect a "pgindent fix for …" follow-up commit.
