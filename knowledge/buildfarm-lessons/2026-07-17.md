## 1. shelduck ‚Äî Configure ‚Äî ICU 55 floor bites SLES 12 SP5 ‚òÖ real, explained

- **Animal:** `shelduck` ‚Äî SUSE Linux Enterprise Server 12 SP5, gcc 4.8.5.
- **Stage:** `Configure` [`32e4508`], 2026-07-17 13:06:30. (Also failed 07-16
  13:26 [`f4f77f5`] ‚Äî same cause, so this is now a persistent master breakage
  for this animal, not a one-off.)
- **Root cause (confirmed from log):**
  ```
  configure:8195: error: Package requirements (icu-uc >= 55 icu-i18n) were not met:
  ```
  Commit `4854b2373c73` "Require ICU 55 or later" (07-16 window) raised the ICU
  floor. SLES 12 SP5 ships an ICU older than 55, so `configure` now hard-fails
  at the `PKG_CHECK_MODULES(icu-uc >= 55)` gate. The many
  `gcc: unrecognized command line option '-Wshadow=compatible-local'` /
  `-Wcast-function-type-strict` lines above it are **normal** configure probe
  failures for an old gcc (4.8.5) and are not the killer.
- **Not a PG bug** ‚Äî it's the intended consequence of the ICU-floor bump. The
  animal owner must either build against a newer ICU or the animal drops off
  `master` (it will still cover the back-branches whose ICU floor it meets).
  This is the concrete downstream cost the ICU-55 commit's discussion
  anticipated. Cross-refs the 07-16 deltas' `#12 4854b2373c73` explainer.

## Takeaway for the corpus

The one durable lesson this window: **a dependency-floor bump (ICU ‚â• 55) is a
buildfarm event, not just a code event.** It silently converts a green
old-distro animal (`shelduck`, and earlier macOS `longfin`/`sifaka`) into a
persistent `Configure`/`Build` red until the animal owner upgrades the
dependency. When the deltas pass flags a "Require X ‚â• N" commit, the buildfarm
pass should expect ‚Äî and did find ‚Äî matching Configure/Build failures on the
lagging animals 12-24h later. No PG-source action; corpus note only.
