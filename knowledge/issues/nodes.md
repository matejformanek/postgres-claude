## Notes

- `read.c` and `params.c` are the two hand-written `.c` files in
  `src/backend/nodes/` that carry non-trivial semantics. The rest of
  the directory (`copyfuncs`, `equalfuncs`, `outfuncs`, `readfuncs`,
  `queryjumblefuncs`) is generated by `gen_node_support.pl` from
  `pg_node_attr` annotations in the header files; issues there go
  upstream of the generator, not in these files.
- The `pg_strtok_ptr` global is the single largest re-entrancy
  hazard in this corner. The current code is safe because no
  readfunc calls user code, but extension writers adding new
  `T_<NodeType>` readers via `extensible.c` could break it.
