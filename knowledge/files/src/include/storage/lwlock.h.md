## Purpose

Public types and protos for the lightweight-lock manager. Defines `LWLock`, `LWLockPadded`, `LWLockMode`, the `MainLWLockArray` layout offsets, and the tranche enum.

### `BuiltinTrancheIds` enum (`lwlock.h:158-173`)

Generated by including `lwlocklist.h` with `PG_LWLOCKTRANCHE(id, name)` expanded to `LWTRANCHE_##id,`. `LWTRANCHE_FIRST_USER_DEFINED` is the boundary for extension-registered tranches.

`LWTRANCHE_INVALID = NUM_INDIVIDUAL_LWLOCKS - 1` — placeholder to initialise the rest from a known base `[from-comment]` (`lwlock.h:160-164`).

## `lwlocklist.h` (related)

Macros `PG_LWLOCK(id, name)` and `PG_LWLOCKTRANCHE(id, name)` declare individual locks and tranches respectively. The file is read multiple times with different macro definitions to generate:
- `BuiltinTrancheNames[]` (`lwlock.c:137-143`).
- `BuiltinTrancheIds` enum.
- Generated `lwlocknames.h` (via `generate-lwlocknames.pl`) defining `NUM_INDIVIDUAL_LWLOCKS` = max id + 1.

Wait-event names in `src/backend/utils/activity/wait_event_names.txt` must stay in sync with tranche names declared here.
