## Role

PG17's templated **adaptive radix tree** (Leis et al., 2013).
Generated by `#define RT_PREFIX foo; #define RT_VALUE_TYPE ŌĆ”;
#include "lib/radixtree.h"` pattern. Replaces vacuum's
`lazy_tid_reaped` bitmap with a far more compact tid store
(`tidstore` in `src/backend/access/common/tidstore.c`), and is
also used by parallel-build sorts and the new shared n-distinct
gather. [from-comment]
`source/src/include/lib/radixtree.h:1-87`

## Public API (generated per prefix)

- `RT_CREATE`, `RT_FREE`
- `RT_FIND(tree, key)`, `RT_SET(tree, key, value_p)`,
  `RT_DELETE(tree, key)` (with `RT_USE_DELETE`)
- Iteration: `RT_BEGIN_ITERATE` / `RT_ITERATE_NEXT` / `RT_END_ITERATE`
- `RT_MEMORY_USAGE`
- DSA mode (`RT_SHMEM`): `RT_ATTACH`, `RT_DETACH`,
  `RT_LOCK_EXCLUSIVE`, `RT_LOCK_SHARE`, `RT_UNLOCK`,
  `RT_GET_HANDLE`
- Required `#define`s: `RT_PREFIX`, `RT_VALUE_TYPE` (or variable
  length via `RT_VARLEN_VALUE_SIZE`/`RT_RUNTIME_EMBEDDABLE_VALUE`)
  [verified-by-code] header lines 89-117.

## Invariants

- INV-1: **Concurrency is caller-managed**: there is a single
  reader-writer lock per tree (`RT_SHMEM` only); writes need
  exclusive, reads need share. The implementation does NOT assert
  that the lock is held by the caller in `RT_SET`/`RT_DELETE` ŌĆö
  only that `tree->ctl->magic` and root pointer are valid.
  [verified-by-code] `source/src/include/lib/radixtree.h:1707-1712`
  (`RT_SET`) and `:2617-2624` (`RT_DELETE`)
- INV-2: keys are unsigned integers up to 64 bits; the tree's
  height is "poor-man's path compression" ŌĆö high zero bytes are
  trimmed. [from-comment] lines 72-76.
- INV-3: with `RT_VALUE_TYPE` Ōēż pointer width and fixed length,
  values are stored inline in child slots (combined
  pointer/value). [from-comment] lines 57-65.

## Notable internals

- Node sizes: 4 / 16 / 32 / 48 / 256 (adaptive switching).
- SIMD (`port/simd.h`) used for node-16 search (line 156).
- The locking TODO (lines 83-87) explicitly notes Optimistic Lock
  Coupling and ROWEX from the "ART of Practical Synchronization"
  paper as future work ŌĆö current impl is **not optimised for high
  concurrency with mixed read-write**.

## Trust boundary (Phase D)

- INV-1 caveat ŌĆö no Assert that the rwlock is held ŌåÆ an
  out-of-tree consumer could call `RT_SET` without exclusive lock
  and silently corrupt the shared tree under load. Internal core
  consumers all wrap with the lock macros, so this is a "trust
  contributors" hazard, not a current exploit.
- Tidstore-on-radixtree is reachable from VACUUM which a user
  cannot directly influence ŌĆö the values stored are TIDs derived
  from the relation being vacuumed.
