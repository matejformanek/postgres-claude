## Role

Internal interface to the **node-tree deserializer** — the inverse of
`outfuncs.c`. Reads a textual serialization (lisp-style sexpr) and
reconstructs a `Node *` tree. Callers go through the public
`stringToNode()` in `read.c`; the funcs here are the implementation
back-end.

## Public API

- `restore_location_fields` flag (`#ifdef DEBUG_NODE_TESTS_ENABLED`,
  `:22-24`) — debug-only round-trip test for parse-tree
  location fields.
- `pg_strtok(int *length) -> const char *` — lisp token parser
  (`:29`).
- `debackslash(token, length) -> char *` — unescape token (`:30`).
- `nodeRead(token, tok_len) -> void *` — recursive entry (`:31`).
- `parseNodeString(void) -> Node *` — top-level deserializer in
  `readfuncs.c` (auto-generated dispatch via
  `gen_node_support.pl`) (`:36`).

## Notable internals

- `parseNodeString` is the auto-generated big switch dispatching on
  the leading `<NODETAG>` token. Per-field reads happen via
  helper macros (`READ_INT_FIELD`, `READ_OID_FIELD`, etc.) defined
  in `readfuncs.c`.
- `debackslash` handles `\\`, `\xNN`, etc.

## Trust boundary / Phase D surface

- **A14-RELATED — DESERIALIZER OF HOSTILE INPUT.** `stringToNode`
  is called from many sites that can ingest untrusted-ish strings:
  - `pg_proc.proargdefaults`, `pg_proc.protrftypes` —
    catalog-stored node trees. A hostile catalog (e.g. corrupt
    pg_dump restore, replication of a malicious upstream) can
    smuggle exotic node-tree shapes.
  - `pg_rewrite.ev_action` — RULE actions.
  - `pg_class.relpartbound` — partition bounds.
  - `pg_index.indexprs`, `indpred` — index expression trees.
  - `pg_attrdef.adbin` — column default expressions.
  - **Plan cache** restored from disk (deserialized at startup).
  - **Parallel-worker plan** transmitted via DSM as a
    serialized string (one of the hottest call sites; runs
    inside parallel workers with the same role as launcher).
- **Trust posture:** the deserializer ASSUMES the source is
  trusted. It does NOT validate that, e.g., a `Var` references a
  valid range-table entry, or that a `Const` Datum length matches
  its type. A poisoned catalog entry can construct a Node tree
  that crashes the planner or executor.
- **No bounds checks on int / pointer fields.** A forged
  serialization with negative array lengths can drive
  `palloc(negative)` → ereport, but in some corner cases drives
  buffer mis-allocation before validation.
- **Logical-replication echo (A8).** Logical decoding does NOT
  ship node trees by default; but custom output plugins might.
  Receiver applying them via `stringToNode` is a new surface.
