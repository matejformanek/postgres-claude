# src/include/common/keywords.h

## Purpose

Public declarations for the SQL keyword tables consumed by the lexer.
Exports the master `ScanKeywords` list plus two parallel arrays keyed
by the position returned by `ScanKeywordLookup`.

## Role in PG

Shared between **frontend** (ecpg, psql tab completion via the
parser-shared code path) and **backend** (the main flex scanner in
`src/backend/parser/scan.l`). The arrays themselves are generated by
`src/tools/gen_keywordlist.pl` from `src/include/parser/kwlist.h`.

## Key declarations

- `UNRESERVED_KEYWORD = 0`, `COL_NAME_KEYWORD = 1`,
  `TYPE_FUNC_NAME_KEYWORD = 2`, `RESERVED_KEYWORD = 3`
  — keyword categories, must match `gram.y`. (`keywords.h:20-23`)
- `extern const ScanKeywordList ScanKeywords` — the master keyword
  list, defined indirectly via `kwlist_d.h` in `keywords.c`.
  (`keywords.h:25`)
- `extern const uint8 ScanKeywordCategories[]` — per-keyword category
  byte, indexed by the lookup result. (`keywords.h:26`)
- `extern const bool ScanKeywordBareLabel[]` — whether each keyword can
  be used as a bare column label (the AS-less form). (`keywords.h:27`)

## Phase D notes

Read-only tables, no trust boundary directly. The lexer uses these to
classify identifiers — if the categories ever got out of sync with
`gram.y`, the parser would mis-classify, but that is a build-time
generator concern (`kwlist_d.h` regen), not a runtime hazard.

## Potential issues

None — pure declarations of generated data.
