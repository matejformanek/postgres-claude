# src/port/pg_crc32c_sse42.c

## Purpose

x86 hardware-accelerated CRC32C implementation using SSE4.2 `_mm_crc32_u*`
intrinsics (the Intel "CRC32" instruction, which despite the SSE4.2 name is
scalar). When `USE_AVX512_CRC32C_WITH_RUNTIME_CHECK` is enabled, also
provides an AVX-512 + VPCLMULQDQ implementation using carry-less multiply
folding over ZMM lanes (generated code from corsix/fast-crc32). Both arms
are gated by a runtime CPUID dispatch in `pg_comp_crc32c_choose` that picks
the best available variant on first call. `[verified-by-code]` `[from-comment]`

## Internal landmarks

- `pg_comp_crc32c_sse42` (`:30`) — eight-byte loop on x86_64 using
  `_mm_crc32_u64`, four-byte on i386 (no 64-bit CRC32 instruction there),
  byte tail. Carries both `pg_attribute_no_sanitize_alignment()` (unaligned
  loads are explicit and safe on Intel) and `pg_attribute_target("sse4.2")`.
  `[verified-by-code]` `[from-comment]`
- `pg_comp_crc32c_avx512` (`:98`) — generated by `corsix/fast-crc32` with PG-style
  whitespace; comment explicitly notes the modifications made (function decl,
  whitespace, alignment threshold). Uses
  `_mm512_clmulepi64_epi128` (`clmul_lo`/`clmul_hi`) for carry-less folding
  across four 128-bit lanes of a ZMM, then a reduction chain back to a
  32-bit CRC via two `_mm_crc32_u64` calls. Threshold of 256 bytes for the
  alignment preamble is documented as "somewhat arbitrary". `[from-comment]`
- `pg_comp_crc32c_choose` (`:171`) — sets `pg_comp_crc32c` to the SB8 fallback
  first (guarded by `USE_SSE42_CRC32C_WITH_RUNTIME_CHECK`), then upgrades to
  SSE4.2 if `PG_SSE4_2` feature is set, then to AVX-512 if both `PG_AVX512_VL`
  and `PG_AVX512_VPCLMULQDQ` are present.

## Invariants & gotchas

- **`_mm_crc32_u*` computes CRC32C (Castagnoli, poly 0x1EDC6F41)**, not the
  more-common CRC32 (poly 0xEDB88320 used in zlib/Ethernet). PG uses CRC32C
  for WAL-record / page-header checksums; mixing up the polynomial would
  silently corrupt validation.
- **Runtime dispatch must run even when compiled for an SSE4.2-capable
  target**, because distro builds target a broad CPU baseline and SSE4.2
  cannot be assumed at runtime on the same binary. Compile-time `__SSE4_2__`
  enables the intrinsic header but does NOT skip the runtime check.
  `[verified-by-code]`
- The AVX-512 arm's alignment preamble (`:105-115`) reads up to 7 bytes
  per-byte before transitioning to 8-byte aligned reads. Buffer must be at
  least `len` valid bytes — no over-read like the popcount AVX-512 arm.
- This file is **not** the dispatcher's choose code on ARM — see
  `pg_crc32c_armv8_choose.c` for that platform's pattern.
