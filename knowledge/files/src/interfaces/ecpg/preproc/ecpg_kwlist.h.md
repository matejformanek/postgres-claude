# `ecpg_kwlist.h` — X-macro table of ECPG-specific keywords

## Purpose

Defines the 42 keywords that are specific to ECPG's `EXEC SQL` command language
and do not appear in the standard backend SQL keyword list. The file is a pure
X-macro data source: it contains no `#ifndef` guard and no definitions of its own.
Each consumer `#define`s `PG_KEYWORD(kwname, value)` before `#include`-ing this
file, then `#undef`s it afterward. The two consumers are `ecpg_keywords.c`
(builds the `ECPGScanKeywordTokens[]` array) and the build tool
`gen_keywordlist.pl` (generates `ecpg_kwlist_d.h` with the perfect hash). The
keywords represent ECPG directives — cursor operations (`open`, `descriptor`),
host-variable modifiers (`indicator`, `reference`), connection management
(`connect`, `disconnect`), error-handling (`whenever`, `sqlerror`,
`sqlwarning`), and C-type aliases (`bool`, `long`, `short`, `signed`,
`unsigned`, `struct`).

## Internal landmarks

- **No include guard** (`ecpg_kwlist.h:18` comment): Intentional — the file is
  designed to be included multiple times with different `PG_KEYWORD` definitions.
  `gen_keywordlist.pl` and `ecpg_keywords.c` both rely on this.
- **ASCII ordering requirement** (`ecpg_kwlist.h:22` comment): `gen_keywordlist.pl`
  requires entries in ASCII order to produce a valid perfect hash. Violating this
  causes a build-time error from the Perl script.
- **Token name convention**: All token names follow `SQL_<UPPER>` (e.g.,
  `SQL_CONNECT`, `SQL_WHENEVER`). These are defined in `preproc.h` (generated
  from `preproc.y` by bison). [verified-by-code, via ecpg_keywords.c:23]
- **C-type keywords present**: `bool` (`ecpg_kwlist.h:26`), `long` (`ecpg_kwlist.h:45`),
  `short` (`ecpg_kwlist.h:56`), `signed` (`ecpg_kwlist.h:57`), `unsigned`
  (`ecpg_kwlist.h:63`), `struct` (`ecpg_kwlist.h:61`) appear here. These differ
  from the C-language keywords in `c_kwlist.h` because ECPG recognizes them in
  a SQL context (e.g., `EXEC SQL BEGIN DECLARE SECTION`), whereas `c_kwlist.h`
  covers them in a C context. [inferred from file content + sibling file review]

## Invariants & gotchas

- **Sorted-list requirement**: The list MUST remain in ASCII order
  (`ecpg_kwlist.h:22`). Adding a keyword out of order silently breaks
  the perfect hash generated by `gen_keywordlist.pl`, producing wrong
  classification at runtime. [verified-by-code, via Makefile + gen_keywordlist.pl contract]
- **No ECPG keyword may duplicate a backend keyword**: Because
  `ScanECPGKeywordLookup` checks the backend table first, any keyword present
  in both tables would always return the backend token value, making the ECPG
  entry unreachable. No duplicates currently exist between this file and
  `src/include/parser/kwlist.h`. [inferred]
- **Token values come from `preproc.h`**, not from a standalone enum. Adding a
  new keyword here requires a matching token definition in `preproc.y` and
  a grammar rule, or the build will fail with an undefined symbol in `keywords.c`
  or `ecpg_keywords.c`. [from-comment in keywords.c]
