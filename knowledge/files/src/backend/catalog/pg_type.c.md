## Purpose

"Routines to support manipulation of the pg_type relation." Universal type-row creation (`TypeCreate`), shell types (`TypeShellMake`), dependency generation, and the array/multirange-name conventions.

## Public surface

- `TypeShellMake` (57) — for CREATE TYPE without a definition (forward declaration); inserts a placeholder pg_type row to reserve the name+OID. Later replaced by full TypeCreate.
- `TypeCreate` (195) — **the universal type inserter.** 25+ params: name, namespace, relationOid (for composite types), relkind, owner, internalSize, typeType ('b'ase, 'c'omposite, 'd'omain, 'e'num, 'm'ultirange, 'p'seudo, 'r'ange), typeCategory, isPreferred, delimiter, in/out/recv/send/typmodin/typmodout/analyze/subscript functions, element type, byval, alignment, storage, typmod, dims, NOT NULL, collation. Returns ObjectAddress. Handles OR REPLACE for shell-→-final and updates pg_depend.
- `GenerateTypeDependencies` (555) — record pg_depend rows for a type: NORMAL on the relation (for composite types), AUTO on the element type (for arrays), AUTO on the range type (for multirange), namespace/owner/I/O functions, extension membership.
- `RenameTypeInternal` (763) — internal-only rename (used when an array type collides with a new composite type).
- `makeArrayTypeName` (838) — `_typname`, truncated to NAMEDATALEN, with `_` suffix incremented on conflict.
- `moveArrayTypeName` (903) — rename a conflicting autogenerated array type out of the way (used by heap.c at CREATE TABLE if the desired rowtype name collides with an existing array type).
- `makeMultirangeTypeName` (948) — `rng_multirange` pattern; called by CREATE TYPE ... AS RANGE.
