# `src/backend/nodes/read.c`

- **Last verified commit:** `e18b0cb7344`
- **Lines:** ~510
- **Source:** `source/src/backend/nodes/read.c`

The S-expression-style deserializer that turns the text produced by
`outfuncs.c` back into live `Node *` trees. Pairs with
`readfuncs.c` (which is mostly generated by `gen_node_support.pl`).
This file owns the low-level lexical tokenizer (`pg_strtok`) and the
upper-level dispatcher (`nodeRead`) — `readfuncs.c` uses both via
`READ_*` macros. [verified-by-code §read.c:300-318]

The serialised form is what gets stored as `pg_node_tree` in catalogs
(check constraints, trigger WHEN clauses, default expressions, view
queries, MV definitions, etc.), shipped across parallel workers, and
written to `pg_rewrite.ev_action`, `pg_proc.proargdefaults`, etc.

## API / entry points

- `void *stringToNode(const char *str)` — primary entry. Internally
  uses static `pg_strtok_ptr`, save/restored so re-entrant calls work.
  [verified-by-code §read.c:89-93, 57-83]
- `void *stringToNodeWithLocations(const char *str)` — only in
  `DEBUG_NODE_TESTS_ENABLED` builds; restores `location` fields rather
  than setting them to -1. Used by the
  `WRITE_READ_PARSE_PLAN_TREES` round-trip test in debug builds.
  [verified-by-code §read.c:95-103]
- `const char *pg_strtok(int *length)` — the tokenizer. Returns
  pointer + length into the source string; never modifies it (unlike
  libc `strtok`). Special tokens are `( ) { }` (each 1 char), `<>` (=
  NULL pointer, length 0), and backslash-escaped runs.
  [from-comment §read.c:112-150]
- `char *debackslash(token, length)` — unescape into a palloc'd
  string.
- `void *nodeRead(const char *token, int tok_len)` — recursive
  reader; external callers pass NULL/0 to read the next token.
  Handles `{NodeType ...}` (via `parseNodeString()` in readfuncs.c),
  the four typed list prefixes (`i`, `o`, `x`, `b` — int / OID /
  TransactionId / Bitmapset), generic node lists, and Value-token
  primitives. [verified-by-code §read.c:319-511]

## Notable invariants / details

- **One global `pg_strtok_ptr`.** Re-entrant `stringToNode` is safe
  because the wrapper saves/restores it on stack, but a partially-
  decoded tree CANNOT call out into something that itself calls
  `stringToNode` interleaved with the current tokenizer's progress.
  Backslash handling is hard-wired (not config'd by terminator set).
  [verified-by-code §read.c:32, 57-83]
- **The four typed-list prefixes** are critical:
  - `(i …)` int list, `lappend_int`
  - `(o …)` OID list, `lappend_oid`
  - `(x …)` TransactionId list, `lappend_xid`
  - `(b …)` Bitmapset (NB: not a `List`)
  - `(<node> …)` generic List of Nodes
  See also `outfuncs.c` and `WRITE_INT_FIELD` macros that emit these
  prefixes. [verified-by-code §read.c:358-458]
- **Number tokens** use `strtoint` for the syntax+range check; on
  ERANGE we fall back to `T_Float` (which just stores the string).
  Floats are stored as strings — never parsed to `double` here.
  [verified-by-code §read.c:255-275, 484-492]
- **Pre-7.0 special case removed.** The pre-7.0 tokenizer treated
  `"..."` as a quoted span; the long NOTE at §read.c:144-150 explains
  the switch to backslash-quoting.
- **`<>` = NULL pointer** — the only way to get a zero-length token.
  Otherwise tokens are at least 1 char. [from-comment §read.c:133-135]

## Potential issues

- **File-line `read.c:32`.** `static const char *pg_strtok_ptr` is
  module-global state. `stringToNodeInternal` saves and restores it,
  but anything that calls into user-provided functions during a
  read (e.g. a Const's input function via the planner?) would be a
  re-entrancy hazard. The current readfuncs.c codepath doesn't do
  that, but it's an implicit invariant worth flagging.
  [ISSUE-undocumented-invariant: pg_strtok_ptr re-entrancy assumes no callout (maybe)]
- **File-line `read.c:482`.** `atoi(token)` is used after
  `nodeTokenType` validated the token via `strtoint`. Using `atoi`
  here loses ERANGE detection; in practice token was already
  range-checked, but the asymmetry between the two parse calls
  (strtoint for type detection, atoi for value extraction) is a
  minor smell. [ISSUE-style: atoi after strtoint validation (nit)]
