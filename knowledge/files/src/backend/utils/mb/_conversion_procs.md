# `src/backend/utils/mb/conversion_procs/` (combined)

- **Last verified commit:** `ef6a95c7c64`
- **Source:** `source/src/backend/utils/mb/conversion_procs/`

Per-conversion encoding shared libraries. Each subdirectory builds a
single small `.so` that exports one or more `pg_*_to_*` C functions
registered as conversion procs via `pg_conversion` catalog rows. They
are loaded on demand by the dynamic-loader (`dfmgr.c`) the first time
a session needs that conversion.

## Structure

Every dir contains a single C file plus optional `*.map`-style headers
generated by `Unicode/UCS_to_*.pl` scripts. The C file is essentially:

1. `#include "../../Unicode/<conv>_to_utf8.map"` for the byte→codepoint
   lookup arrays.
2. Glue function calling `LocalToUtf` / `UtfToLocal` from `conv.c` with
   the right table pointer + radix.

## Directory map

- `cyrillic/` — koi8r ↔ iso8859-5 ↔ win1251 ↔ alt direct pairs.
- `euc2004_sjis2004/`, `euc_jp_and_sjis/`, `euc_tw_and_big5/` — direct
  CJK pairs that don't go through UTF-8.
- `latin2_and_win1250/` — Polish/Czech/Slovak pair.
- `utf8_and_*` (the vast majority) — UTF-8 ↔ legacy encoding, one
  dir per family.

## Map data

Maps live in `src/backend/utils/mb/Unicode/`; the `.pl` scripts there
download authoritative tables from unicode.org and emit C arrays.
Edits should re-run the scripts, not hand-edit the generated files.

## Loading

- `pg_conversion` rows reference these by `proname`.
- `dfmgr.c` resolves the `.so` path; first call into the function loads
  it via `dlopen` and pins it for the backend's lifetime.
- Conversion procs are bog-standard fmgr V1 functions of the shape
  `(integer, integer, cstring, internal, integer, boolean) → integer`
  returning the number of input bytes consumed.
