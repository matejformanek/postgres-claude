Vendored copy of the CPAN module `Devel::PPPort`'s portability header
(generated automatically by the `Devel::PPPort` toolchain — the file
opens with a Perl `=pod`/`SKIP` self-extracting preamble and the
banner `Perl/Pollution/Portability Version 3.63`). Provides the macro
shims, function back-ports, and API-rename guards that let
`plperl.c` and `plperl_system.h` use the modern Perl XS API while
still compiling against the wide range of Perl interpreters PG
supports on the build host. **Not PostgreSQL code** — do NOT
hand-edit; replacement happens by re-running
`perl -MDevel::PPPort -eDevel::PPPort::WriteFile` against a newer
CPAN release. [verified-by-code: `source/src/pl/plperl/ppport.h:1-22`,
file header banner]

## Why it's in the tree

PL/Perl is a backend extension that calls into `libperl` via the XS
API (`Perl_xxx`, `SV*`, `HV*`, `AV*`, `GV*`, `dTHX`, etc.). The XS
API has churned across Perl 5.6 → 5.10 → 5.14 → 5.20 → 5.28 → 5.36 →
5.38. `ppport.h` papers over the churn by providing missing macros
(`HeUTF8`, `GvCV_set`, `newSVpvn_flags`, `SvREFCNT_inc_simple_void`,
…) on old Perls and no-ops on new ones, so `plperl.c` can target
"sufficiently recent XS" without `#ifdef PERL_VERSION` ladders.
[inferred from `source/src/pl/plperl/plperl_system.h:176-189` which
includes `ppport.h` then adds two PG-side overrides for `HeUTF8` and
`GvCV_set` it says "ppport.h doesn't supply"]

## API surface (high level)

The file exposes hundreds of feature-test macros named
`NEED_<symbol>` / `NEED_<symbol>_GLOBAL`, plus the actual shim
definitions. Typical patterns it covers (sample, NOT exhaustive):

- Missing accessors: `SvPV_nolen`, `SvPV_const`, `SvPVbyte_nolen`,
  `SvUTF8`, `HeUTF8`. [from-comment, header banner reference]
- New constructors back-ported: `newSVpvn_flags`, `newSVpvn_share`,
  `newRV_noinc`, `sv_2pv_flags`. [from-comment]
- Threading macros: `aTHX`, `pTHX`, `dTHX`, `dTHXa` — no-ops on
  unthreaded Perls. [from-comment]
- `Perl_xxx` underscore-namespace renames between versions.
  [from-comment]
- Warning-category constants: `WARN_ALL`, `packWARN`, `Perl_warner`.
  [from-comment]

Nothing in this list is PG-defined — every symbol is documented in
`perldoc Devel::PPPort` (also embedded in the file's POD block,
`source/src/pl/plperl/ppport.h:1-460` approx).

## Notable invariants / details

- **Self-modifying file.** The opening lines (`#if 0`, `my $void =
  <<'SKIP';`, `#endif`) make ppport.h simultaneously a valid C
  header AND a valid Perl script that, when run, regenerates itself
  from a newer `Devel::PPPort`. The
  `=head1 NAME` POD block (line ~22 onward) is C-skipped by the
  `=pod...=cut` syntax which C ignores via `#if 0` framing.
  [verified-by-code: `source/src/pl/plperl/ppport.h:1-3`]
