# Deep-copying a `Node *expr`

## Tool

`copyObject(expr)`. Macro defined in `src/include/nodes/nodes.h:228-233`:

```c
#ifdef HAVE_TYPEOF_UNQUAL
#define copyObject(obj) ((typeof_unqual(*(obj)) *) copyObjectImpl(obj))
#else
#define copyObject(obj) copyObjectImpl(obj)
#endif
```

`typeof_unqual` lets the macro preserve the argument's pointer type — `copyObject(expr)` returns `Expr *` (or whatever the input was), no cast needed.

## What it does under the hood

`copyObjectImpl` lives in `src/backend/nodes/copyfuncs.c:177-212`. It:

1. `check_stack_depth()` at the top (copyfuncs.c:185) — bails with the "stack depth limit exceeded" error if you've got a pathological recursive parsetree. This is the same hand-rolled stack guard that walkers/mutators use.
2. Switches on `nodeTag(from)` and dispatches to `_copyFoo()`. The per-node bodies in `copyfuncs.funcs.c` are auto-generated by `gen_node_support.pl`; manual overrides for the special-cased nodes (`Const`, lists, extensible) live in `copyfuncs.c`.
3. Returns a freshly-palloc'd tree in `CurrentMemoryContext`. The caller decides which context that is — if you want the copy in a longer-lived context, `MemoryContextSwitchTo(longlived)` before the call.

## Gotchas

- **By-ref Datums** (varlena, cstring, fixed-length-byref): handled by `_copyConst` → `datumCopy(value, typByVal, typLen)` in `utils/adt/datum.c`. The Datum field is deep-copied so you can scribble on it safely. If you build your own `Const` outside the copy machinery and forget `datumCopy`, you alias the original. By-val Datums (int, oid, float8 on 64-bit) are copied trivially.
- **List flavors**: `T_List` is deep-copied via `list_copy_deep` — every element is `copyObject`'d. But `T_IntList` / `T_OidList` / `T_XidList` are shallow `list_copy` only (the scalar elements have no internal structure to recurse into). Be aware if you stash node pointers in an `OidList` (don't — wrong list type).
- **Extensible nodes** (`T_ExtensibleNode`): `_copyExtensibleNode` looks up the registered `ExtensibleNodeMethods` and dispatches to the extension's `nodeCopy` callback. If you forgot to call `RegisterExtensibleNodeMethods()` at module load, you get a runtime error.
- **Stack depth**: very deep expression trees (e.g. a 10 000-way OR) blow the stack guard before they OOM. There is no iterative fallback.
- **Custom copy bodies**: a struct annotated `pg_node_attr(custom_copy_equal)` skips the generator and the body lives in `copyfuncs.c` proper — `nodetag_only` opts out of copy/equal entirely (executor state nodes).

## The round-trip

`copyObject(stringToNode(nodeToString(p)))` is the parse-and-plan-tree integrity check exposed via `debug_copy_parse_plan_trees` / `debug_write_read_parse_plan_trees` GUCs. It's also load-bearing for: plan cache (a cached plan is re-copied per execution), rule storage (`pg_rewrite.ev_action` is `nodeToString`'d at CREATE RULE and `stringToNode`'d at rewrite), and parallel-worker plan shipping (the shared-memory FSM ships `nodeToString` over DSM).
