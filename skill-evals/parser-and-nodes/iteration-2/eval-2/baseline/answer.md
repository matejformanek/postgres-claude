# Deep-copying a `Node *expr`

## Tool

Use `copyObject(p)`. It's a macro defined in `src/include/nodes/nodes.h` that wraps `copyObjectImpl(const void *)`. On compilers that have `typeof_unqual` (most modern GCC/Clang), the macro casts the result back to the argument's pointer type so you don't need an explicit cast.

## What it does

`copyObjectImpl` (in `src/backend/nodes/copyfuncs.c`) is a big switch on `nodeTag` that dispatches to a per-node `_copyFoo()` function. Most of those are auto-generated by `gen_node_support.pl` from the struct definitions in `parsenodes.h` / `primnodes.h` / `plannodes.h`. The result is a fresh deep copy allocated in `CurrentMemoryContext`.

A `check_stack_depth()` call protects against blowing the C stack on pathological recursion (very deep ORs, etc.).

## Gotchas

- **By-ref Datums**: `_copyConst` calls `datumCopy()` to actually duplicate the underlying bytes for varlena/by-ref types. By-val Datums are copied trivially.
- **Extensible nodes**: `T_ExtensibleNode` dispatches to a registered method table — the extension must register its copy callback at module load.
- **List flavors**: `List` of `Node *` is deep-copied. `IntList` / `OidList` / `XidList` (lists of scalars) only need shallow copies.
- **Memory context**: the copy lands in whatever `CurrentMemoryContext` is when you call. If you want it in a query-lifetime context (e.g. `MessageContext`), switch first.
- **Stack depth**: deeply nested expression trees hit the stack guard. No iterative fallback.

## When to use what

- Deep copy → `copyObject`.
- Structural compare → `equal(a, b)` (`equalfuncs.c`).
- Serialize → `nodeToString` / deserialize → `stringToNode`. Used by the plan cache, rule storage, and parallel worker plan shipping.
