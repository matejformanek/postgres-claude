### with_skill (iter-2)

#### `## Verdict`

**REJECT-A** — all critical design problems identified; correct
alternative proposed. Grade per
`.claude/skills/review-checklist/SKILL.md` Phase 0 rubric, summarized
in `pg-feature-plan/SKILL.md` REJECT-track section.

#### Reasons (5 numbered)
1. **Network I/O inside `money_out()` violates immutability invariant.**
   `cash_out` is the type-output function and is called from every
   `SELECT money_col` in the executor's per-tuple loop. Cite:
   `source/src/include/catalog/pg_proc.dat` `cash_out` row. Persona
   reflex: Tom Lane — type-system + dump-determinism
   (`knowledge/personas/tom-lane.md`).
2. **Backend `Datum` functions don't do outbound HTTP.** No timeout
   policy, no DNS sandbox, no retry semantics, no security model.
   Persona reflex: Noah Misch — security
   (`knowledge/personas/noah-misch.md`).
3. **Non-deterministic `*_out()` breaks logical decoding + dump/restore.**
   Per `knowledge/subsystems/replication.md`, type-output functions
   must be deterministic. Cite: subsystem doc's INV-tag for
   `output_plugin` callback contract.
4. **`money` is deprecated guidance.** Docs recommend `numeric` for
   currency; deepening `money` runs counter to direction-of-travel.
5. **No upgrade path / no test surface.** `pg_dump` reproducibility is
   broken; no test demonstrates determinism.

#### Predicted lead reviewer

**Tom Lane** — type-system + dump-determinism reflex. Secondary:
**Andres Freund** (per-row hot-path; lockfuncs.c-adjacent area committer).

#### Concrete alternative

`money_in_currency(money, text) returns text` — immutable user-callable
function taking an explicit rate parameter or joining against a
user-managed rates table. Keeps `money_out()` pure.
