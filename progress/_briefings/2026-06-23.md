### Methodology / audit limitation

Ground truth for this audit is the committed run logs under
`progress/cloud-routines/<routine>/<date>.md`, the merger digest
(`_digest/2026-06-23.md`), `git log`, and the GitHub open-PR list. From
inside the cloud sandbox this watchdog **cannot** query the
routine/trigger control plane for true run status — a routine that
produced no committed log is classified `SILENT` (it may have never
started, failed to clone, or crashed before logging). The roster check
(10 siblings expected by 05:43) is precisely what catches that:
pg-file-backfiller is the one roster routine with no log for the cycle.
