## Tooling note (for pg-state-keeper)

The recipe's primary source URLs point at `api.github.com` / `github.com`,
which in the cloud CCR session are **GitHub-MCP-scoped** to the meta repo and
return 403 ("GitHub access to this repository is not enabled for this session")
for postgres/postgres; unauthenticated `WebFetch` hit GitHub's 429 secondary
limit. Workaround used: `git.postgresql.org` gitweb (RSS + `commitdiff_plain`)
and `buildfarm.postgresql.org` are reachable via plain `curl` (200) and are a
complete substitute. Recommend the recipe document git.postgresql.org as the
canonical fetch path for cloud runs, since `GH_TOKEN` does not lift the
repo-scoping on the postgres/postgres origin.
