  These are invisible to the mechanical Pass 3 mirror because it can't derive
  which register a tag belongs to without a pointer. Recipe step 9 permits
  creating a register from `_template.md`, but reliably deriving the parent
  subsystem for 145 docs mechanically is error-prone and risks stepping on
  auditor-curated territory (R7 scope discipline) — prior runs consistently

  resolver (from the per-file doc's `## Cross-refs` block) to close this gap
  deterministically, or the pg-quality-auditor could add register pointers when
  it curates the tags.

  backtick-frequency is dominated by SQL keywords, corpus-structural tokens,
  and file/module basenames; only after filtering those does the real-internals
  seam surface. Consider baking the filter (SQL-keyword set + a

  re-derive it. The remaining defensible seam after this run is thinner — expect
  the next few runs to land in the +40-70 range as the high-frequency internals
  tail is consumed.
