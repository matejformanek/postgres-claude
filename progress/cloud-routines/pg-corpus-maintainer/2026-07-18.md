## Follow-ups surfaced (out of scope — not fixed here)

- **Pass 1 gate is a latent under-reconciliation bug.** Prior runs gate Pass 1 on
  a long-form-doc git diff; because `pg-corpus-graph-refresh` grows the
  forward-edge tables on its own cadence, backlog re-accumulates. Durable fix:
  always run the (cheap, idempotent) reverse-index rather than gate it. This run
  proves the reconciliation is safe (614/614 forward-edge-backed, 0 orphans).
- **Issue-register format split.** 7 contrib/include registers (`sepgsql`,
  `hstore`, `ltree`, `sslinfo`, `postgres_fdw`, `include-storage`,
  `include-common`) use a narrative `### <file>` layout with empty `Open/Triaged`
  tables; the rest use the tabular layout. A one-time normalization would let
  Pass 3 mirror uniformly, but must not be done as a nightly churn.
- **Prose-embedded issue line numbers.** 3111 of 3262 `[ISSUE-*]` tags carry their
  file:line only in the bullet prose ("Line NNN…"), not inline in the tag. A
  deterministic per-doc structural parser would let Pass 3 reach them; error-prone
  to do mechanically without stepping on auditor-curated registers.
