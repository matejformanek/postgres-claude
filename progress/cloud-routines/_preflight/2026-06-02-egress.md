## Proxy header detail

The `x-deny-reason: host_not_allowed` header (seen on all non-GitHub 403s) confirms that an
outbound HTTP proxy enforces an allowlist. Blocked hosts receive an immediate HTTP/2 403 with
`content-type: text/plain` and `content-length: 21`; the TCP connection itself succeeds, so
this is a deliberate policy block — not a DNS failure or firewall RST.

GitHub hosts (`api.github.com`, `raw.githubusercontent.com`) pass through without that header,
confirming they are on the allowlist.
