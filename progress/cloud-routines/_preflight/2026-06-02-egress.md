## Analysis

The network policy uses an **allowlist** model. Blocked hosts return HTTP 403 with  
`x-deny-reason: host_not_allowed` — this is the egress filter, not the origin server.  
The only confirmed-reachable hosts are:

- `api.github.com` — REST API fully reachable; unauthenticated rate limit (60/hr) exhausted at time of test
- `raw.githubusercontent.com` — fully reachable; `README.md` returned 200

All other hosts tested — including the neutral control host `example.com` — are blocked  
by the network policy before reaching the origin.

The GitHub MCP server tools (mcp__github__*) are available and authenticated via  
a separate credential mechanism (not the shell-visible env vars), confirming that  
programmatic GitHub access works through the MCP layer regardless of egress policy.
