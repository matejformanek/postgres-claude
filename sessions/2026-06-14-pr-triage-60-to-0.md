## Method

### Phase 1 — code-review

Spawned 3 general-purpose Agent subagents in parallel, each handling
~20 PRs. Per-PR checks:

1. **Anti-target audit**: `git diff --stat origin/main..HEAD --
   knowledge/calibration knowledge/personas knowledge/files
   patches progress/STATE.md progress/cloud-routines CLAUDE.md
   pg-claude-plan.md` MUST be empty (with cloud-branch exemption).
2. **Class detection**: branch prefix → validation track.
3. **Supersession**: any newer PR with same purpose?
4. **Cite spot-check**: random sample of 1-2 file:line cites
   resolved against `source/` at `e18b0cb7344`.
5. **Established shape**: anchors block + invariants + greps +
   cross-references present (corpus PRs only).

Each agent returned a verdict table (`#N CLASS VERDICT ACTION NOTES`)
plus a one-paragraph batch summary.

### Phase 3 — Companion fix

PR #239 (`ft_cloud_merger_all_prs`) extends the `pg-evening-merger`
recipe from `head:cloud/`-only to ALL open PRs. Per-class
validation tracks defined:

- **cloud** — existing behavior preserved (run-log + queue marker +
  cited URLs).
- **corpus** — anti-target audit + cite-resolve + cross-ref resolve
  + established-shape + ≤4-files scope + R10 separation.
- **skills** — frontmatter + companion_skills + benchmark
  non-regression.
- **session-log** — single-file + dated + no protected paths.
- **feature** — R10 separation enforcement (no source/ edits).
- **hotfix** — single-purpose.
- **unclassified** — comment + leave open.

Plus a "superseded-PR auto-close" rule for the routines whose
output is one-per-day (anchor-refresh, state-keeper,
corpus-maintainer).

This is the root-cause fix: without it, the next mining arc would
produce the same backlog.
